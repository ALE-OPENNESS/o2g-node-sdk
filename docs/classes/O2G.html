    Class O2G

    Main entry point for the O2G SDK.

    The `O2G` class is a static facade that manages the application lifecycle and provides access to all O2G services. The typical usage follows three steps:

    1. Initialize the application with O2G.initialize
    2. Login with O2G.login
    3. Optionally subscribe to events with O2G.subscribe

    Example

    O2G.initialize("MyApp", { privateAddress: "192.168.1.1" });
    const success = await O2G.login("user", "password");
    if (success) {
        await O2G.subscribe(subscription);
        const calls = await O2G.telephony.getCalls();
    }

    Staticinitialize

    • initialize(appName: string, host: Host, apiVersion?: string): void

      Initializes the O2G application with the given name, host, and optional API version.

      This method must be called before any other method. It can only be called once; subsequent calls will throw an error.

      Parameters

      • appName: string

        the application name, used to identify this client on the O2G server

      • host: Host

        the O2G server host configuration

      • apiVersion: string = '1.0'

        the API version to use. Defaults to "1.0"

      Returns void

      Throws

      if the application has already been initialized

          loginName: string,

          supervisedAccount?: SupervisedAccount | null,

      Authenticates the user against the O2G server and initializes all services.

      Must be called after O2G.initialize. On success, all service getters become available. On failure, returns `false` and logs the error.

      Parameters

        the user's login name

      • password: string

        the user's password

      • supervisedAccount: SupervisedAccount | null = null

        optional supervised account to open the session on behalf of another user

      Returns Promise<boolean>

      true if login succeeded; false otherwise
